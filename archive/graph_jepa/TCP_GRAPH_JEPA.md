# TCP-Graph-JEPA - archived graph-based anomaly-detection exploration

> Archived side quest, self-contained under `archive/graph_jepa/v2/`. It is
> not evidence for the retained two-view SIGReg/PEIRA EEG pipeline.

## Idea in one paragraph
The 22 **TCP bipolar derivations** (TUH `01_tcp_ar` montage) are treated as the
**nodes of a graph**. We pretrain a JEPA on (by default) *normal* EEG only: mask
channel-time tokens, and predict the **masked latent embeddings** from the
visible spatial-temporal graph context. We never reconstruct the raw signal — the
target is the latent of an EMA **target encoder** (stop-gradient), JEPA-style.
At inference, the **latent prediction error** is the anomaly score: an event is
abnormal when it is *not predictable* from its spatial-temporal neighbourhood.
Window scores aggregate to a file-level normal/abnormal score, and the
per-position error is an interpretable **channel × time heatmap**.

## Why a graph
Edges encode the two priors a clinician uses:
- **shared-electrode** edges — derivations sharing a physical electrode are
  scalp-adjacent (`Fp1-F7` ↔ `F7-T3`, … ; auto-generated by electrode overlap);
- **contralateral "brain-twin"** edges — left/right homologues (`Fp1-F7` ↔
  `Fp2-F8`, …); abnormality is often a *break* of left-right symmetry, so an
  explicit edge lets the model predict one side from the other;
- **self-loops** — keep a node's own state.

`build_tcp_graph` returns a PyG-compatible `edge_index` + `edge_type`
(`0=shared, 1=contralateral, 2=self`); `build_dense_adjacency` returns the dense
per-relation matrices used by the in-repo message passing (no PyTorch-Geometric
dependency).

## Architecture (`archive/graph_jepa/v2/core/model.py`)
1. **feature projection** `Linear(F → H)`
2. **temporal encoder** per channel over time (Transformer / GRU / TCN)
3. **graph encoder** dense **relational** message passing over the 22 nodes at
   each time step: `h' = LN(h + Σ_r A_r·(h W_r))` then a feed-forward block
4. **target encoder** EMA copy of the context encoder, stop-gradient
5. **predictor** MLP that maps the context latent at masked positions onto the
   target latent

Loss (masked **and** available positions only):
`loss = smooth_l1( predictor(context(x_masked))[M] , sg(target(x))[M] )`.

## Data → `[C, T, F]`
`archive/graph_jepa/v2/core/windows.py` builds the 22 bipolar channels from referential
electrodes (`EEG FP1-REF` …) and extracts **log-bandpower** features
(`features.py`): 5 bands (δ θ α β γ), default 7 s windows at 0.1 s frames →
`[22, 70, 5]`. Missing electrodes are handled gracefully — TUAB's `01_tcp_ar`
files have no A1/A2, so `A1-T3` and `T4-A2` are zero-filled and flagged
**unavailable** (never masked, never scored). Sources: `edf`, `tensor`
(pre-extracted `[N,C,T,F]`), `synthetic`. Feature normalisation uses
**train-only** statistics (no leakage), stored in the checkpoint.

## Masking (`masking.py`)
`random` channel-time tokens · `channel` (whole channels over a span) ·
`contralateral` (mask one hemisphere, predict from the other) · `mixed`.
Default ratio 0.25.

## Anomaly scoring (`scoring.py`)
Per window we mask many times so every position is hidden at least once and
average the error into a dense `[C, T]` heatmap. Aggregations (configurable):
`mean` / `max` / `top_k_mean` (default = mean of top-10% errors). Window scores →
file score the same way.

## Metrics (`metrics.py`)
With abnormal labels: **AUROC, AUPRC, balanced accuracy, F1, confusion matrix**;
threshold via Youden's J. Normal-only regime: conformal-style
`threshold = quantile(normal_val_scores, 0.95)`.

## Commands
```bash
# train (SSL on normal TUAB windows)
python -m archive.graph_jepa.v2.scripts.train_graph_jepa \
    --config archive/graph_jepa/v2/config.yaml data.data_root=<TUAB_PREPROCESSED>

# evaluate (file-level normal vs abnormal on the eval split)
python -m archive.graph_jepa.v2.scripts.evaluate_graph_jepa \
    --ckpt ./checkpoints/graph_jepa/latest.pth.tar --data-root <TUAB_PREPROCESSED>

# visualise one recording -> channel×time heatmap + timeline + JSON/CSV
python -m archive.graph_jepa.v2.scripts.visualize_graph_jepa \
    --ckpt ./checkpoints/graph_jepa/latest.pth.tar --edf <file.edf> \
    --output-dir ./archive/graph_jepa/v2/results/viz_local [--contiguous]

# classical baselines (IsolationForest / OneClassSVM on bandpower)
python -m archive.graph_jepa.v2.scripts.baselines --data-root <TUAB_PREPROCESSED>

# offline smoke (no data) and unit tests
python -m archive.graph_jepa.v2.scripts.smoke_graph_jepa
python -m pytest archive/graph_jepa/v2/tests -q
```

Any config leaf is overridable on the CLI, e.g. `model.hidden_dim=256
mask.mode=contralateral data.window_sec=10`.

## Results — real TUAB run on Dalia (B200)
SSL pretrained on **normal** TUAB windows only (15 epochs, ~3 s/epoch), evaluated
on the full eval split (**150 normal / 126 abnormal**, file level):

| metric | value |
|---|---|
| **AUROC** | **0.791** |
| **AUPRC** | **0.780** |
| balanced acc (Youden threshold) | 0.741 |
| balanced acc (conformal q=0.95)  | 0.700 |
| F1 (Youden) | 0.688 |

**Key finding — the direction is inverted.** The naive JEPA prior is "anomaly =
high latent error", but on TUAB the *abnormal* class has **lower** error
(raw AUROC 0.209). Abnormal EEG (pathological slowing / rhythmic, spatially
redundant) is **more** latent-predictable than the richer, less stereotyped
normal EEG. So the discriminative score is the *negative* JEPA error; we calibrate
this single sign bit on labels (`metrics.orient_scores`) and report the oriented
AUROC = `max(auroc, 1-auroc) = 0.791`. The `tgt_std` diagnostic logged each epoch
stays well above zero, so this is a genuine signal, **not** representational
collapse (which would give AUROC ≈ 0.5).

Context: this is **fully unsupervised** (no labels in pretraining; one polarity
bit at eval) and lands in the same band as the *frozen* foundation-model numbers
the team cites (BIOT frozen 0.78, EEG2Rep 0.832 AUROC) and the project's own
supervised linear probe (~0.82). Honest framing for the jury: a clean,
interpretable, label-free detector with a non-obvious mechanistic finding
(abnormal = more predictable), not a SOTA claim.

Reproduce: `sbatch archive/graph_jepa/v2/cluster/graph_jepa.sbatch` then
`evaluate_graph_jepa --threshold-from {normal,youden}`. Artifacts in
`archive/graph_jepa/v2/results/` (`eval_metrics_oriented.json`, `viz_*/heatmap.png`).

## Acceptance / smoke
A synthetic test injects a high-amplitude block into one channel-time region and
confirms the anomaly score there exceeds the window average (latest local run:
≈3.8×). The model forward accepts `x:[B,22,T,F]`; the graph builder returns valid
22-node edges; training loss decreases on a small overfit subset.
